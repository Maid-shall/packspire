# PACKSPIRE ゲームデータ・実装仕様

このファイルは、現在実装されている全システムと調整値の正本です。ゲーム概要と遊び方は `README.md`、未実装構想は `UNIMPLEMENTED_IDEAS.md` を参照してください。

## 実装済みシステム索引

- 36区画オープンダンジョン、移動敵・商人、警戒段階、地図断片、途中帰還
- バックパック色マス、上辺カード化、同色追加効果、装備隣接LINK、安全マス、ドラッグ移動・個別取り外し
- 保管装備、家宝認定、持込装備、支給品、戦果選択、保険、傷跡
- 複数役職レベル、上級・複合・隠し職、勢力限定職
- 勢力評判、外交係数、勢力関係図、階級能力、情勢フラグ
- キャラクター作成、役職衣装、勢力別拠点、図鑑、記憶庫

このファイルは、ゲーム内の数値と条件を調整するための一覧です。実際の値は [`game.js`](game.js) に定義されています。この資料を変更しただけではゲームには反映されないため、調整時は記載された定義名も同時に変更してください。

## 装備一覧

### 装備個体データ

同じ種類の装備でも別個体として扱います。所持品には装備ID文字列ではなく、次の情報を持つ個体オブジェクトを保存します。

| 項目 | 内容 |
|---|---|
| `uid` | 個体を識別する一意ID。配置や生成カードもこのIDを参照 |
| `templateId` | `ITEMS` に定義された装備種類ID |
| `durability` | 個体ごとの現在耐久度。初期値6 |
| `uses` | その個体から生成されたカードの累計使用回数 |
| `heirloom` | 将来の家宝指定に使うフラグ |
| `evolution` | 将来の個体進化・変質に使うデータ |
| `scars` | 将来の傷・恒久的特徴を記録する配列 |
| `insured` | 将来の保険効果に使うフラグ |

旧形式の装備ID文字列は読み込み時に自動で個体へ変換されます。永続データのスキーマ版は `version: 8` です。装備・拠点・ダンジョン・役職・勢力に加え、作成したキャラクター外見を保存します。

## キャラクター作成と拠点表示

- 初回起動時にキャラクター作成画面を表示します。
- 体格・顔は男性系2種、女性系2種の合計4プリセットです。能力差はありません。
- 各プリセットに短髪、ミディアム、結び髪の3種があります。
- 選択内容は `META.character.body` と `META.character.hair` に保存します。
- キャラクター素材は4列×3行の透過済み `assets/character-creator-sheet.png` です。
- 役職衣装は4体格×4系統の透過済み `assets/role-costume-sheet.png` を素体へ重ねます。
- 緑背景はCanvas描画時にクロマキー処理し、透過表示します。
- 拠点中央と戦闘画面のプレイヤー表示で同じ外見を使用します。
- 拠点では現在職に応じた衣装とオーラ、役職名・レベル、勢力階級、選択中の家宝1個を表示します。
- 勢力背景は `assets/faction-hub-sheet.png` の2×2差分です。

形状は左上を `[0,0]` とした相対座標です。回転時は90度ずつ変化します。

### 装備調整時の注意

- 形状は `ITEMS.<ID>.shape` で変更します。
- 配置効果の説明文は `rule`、実際の計算は `deckData()` 内にあります。両方を変更してください。
- `type` は `weapon`、`armor`、`rune`、`supply` のいずれかです。役職・配置効果・隠し職判定に使われます。
- 装備イラスト位置は `ITEM_ART` に定義されています。

### 複数役職レベル

- 役職レベルは `META.jobLevels` に恒久保存されます。
- 初回のみ役職を選択します。現在職は `META.currentRole` に保存され、次回以降は選択画面を飛ばします。
- 役職変更はダンジョン内の「継職の祭壇」でのみ可能です。戦闘勝利と祭壇修練で現在職のレベルを獲得します。
- 基本職と上級職の最大レベルは10、隠し職は15です。
- 全役職を合計したキャラクターレベルは100が上限です。
- 習得済み役職の戦闘効果はすべて同時に適用されます。
- 数値補正の追加成長は3レベルごとです。複数職を重ねても1レベルごとに急激に強くならないよう抑えています。
- 基本職Lv.5以上で対応する上級職を祭壇から解禁できます。転職しても元の役職レベルは失われません。

既存の配置・装備条件に加え、次の役職レベル構成でも隠し職を解禁できます。

| 隠し職 | 役職レベルによる代替解禁条件 |
|---|---|
| 万刃の王 | 戦士Lv.5＋斥候Lv.5 |
| 不動の聖者 | 守護兵Lv.5＋工匠Lv.5 |
| 境界の織手 | 工匠Lv.7＋斥候Lv.3 |
| 盤上舞踏家 | 四基本職がすべてLv.3以上 |

## 協力勢力と階級

鉄殻軍、胞子教団、荷造り師組合は初期解禁です。遠征準備で協力勢力を選び、戦闘勝利ごとに貢献度+1、ボス勝利時は+3を得ます。虚無の巡礼者は専用イベント、または虚ろなる大記憶庫踏破で解禁されます。

| ID | 勢力 | 階級効果 |
|---|---|---|
| `iron` | 鉄殻軍 | 防具カードの防御+階級 |
| `spore` | 胞子教団 | 戦闘勝利後の追加回復+階級 |
| `guild` | 荷造り師組合 | 商店価格を階級×5%割引 |
| `void` | 虚無の巡礼者 | 獲得金を階級×5%増加、呪剣が報酬候補に出現 |

| 階級 | 必要貢献度 | 限定職 |
|---:|---:|---|
| 1 | 0 | なし |
| 2 | 10 | なし |
| 3 | 25 | 勢力ごとの限定職を解禁 |
| 4 | 50 | 最高効果 |

階級3で鉄盟前衛、胞子森導師、荷造商務官、虚無の使徒をそれぞれ解禁します。勢力はジョブとは別の成長軸で、勢力を変更しても過去の貢献度は失われません。

## 商店価格

基本価格は次の式です。

```text
14 + 使用マス数 × 4
```

補正は乗算されます。

- 行商人の鞄：×0.8
- 荷造り師組合：×0.85

例：2マス装備を行商人の鞄＋組合階級3で買う場合、`(14+8)×0.8×0.85=14.96` を四捨五入して15Gです。

### 色マス・上辺カード化・装備LINK

バックパックの各マスには火・水・風・土の色があります。装備にもセルごとの色と◆取っ手があり、◆がバックパックの上辺へ触れた装備だけがカードを生成します。装備セルと真下のバックパックマスが同色なら、その装備自身へ単純な追加補正が発生します。回転すると形状・取っ手・装備色が一緒に回転します。

色一致とは独立して、上下左右に接する装備同士はLINKします。剣と盾、ルーンと武器、結晶と装備などの固有LINK効果があり、剣＋熾火、盾＋結晶、爆弾＋フラスコはカードを追加せず上位カードへ置換します。色一致だけを狙うか、上辺へ出すか、複数装備を隣接させるかが競合する構成です。

| 属性条件 | 基本効果 |
|---|---|
| 火1点ごと | 攻撃+2 |
| 水1点ごと | 防御・回復+2 |
| 風1以上 | コスト-1 |
| 土1点ごと | 生成カードへ防御+2 |

属性は装備ごとに一致したセルだけを集計します。カードの特殊変化は色一致ではなく装備同士のLINK条件で発生します。装備が一つもカード化されていなくても、現在の基本職に応じた4枚の職業カードで戦闘を開始できます。

## バランス調整の推奨手順

1. まず基礎カードのダメージ・防御・コストを調整する。
2. 次に配置効果なしの初期装備で各敵を試す。
3. 配置シナジーを最大化した場合の上振れを確認する。
4. 役職Lv.1とLv.3、上級職で同じ戦闘を比較する。
5. 契約を複数付けた場合の報酬倍率と難易度を確認する。
6. 恒久強化が最大の状態でもボスが数ターンで無条件撃破されないか確認する。

### 現在の注目点

- 0コストの《刺突》《受け流し》《整流》は、役職や結晶で強化されると連打性能が高くなります。
- 工匠系役職＋星喰い爆薬は大きく上振れするため、ボス部位報酬との組み合わせを確認してください。
- 城塞騎士は防御カードすべてへ補正するため、レベル上昇時に無敵化しやすい構造です。
- 焦燥＋封印契約は獲得金1.95倍になるため、商店経済が崩れないか確認してください。
- 呪剣は配置必須・取り外し不可・隣接コスト増加があります。実際の不利益が十分か確認してください。
- 部位攻撃中は本体HPを削らないため、部位報酬がリスクに見合うか確認してください。
