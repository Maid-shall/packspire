# PACKSPIRE ゲームデータ・実装仕様

このファイルは、現在実装されている全システムと調整値の正本です。ゲーム概要と遊び方は `README.md`、未実装構想は `UNIMPLEMENTED_IDEAS.md` を参照してください。

## 実装済みシステム索引

- 36区画オープンダンジョン、移動敵・商人、警戒段階、地図断片、途中帰還
- 個体ランダム色・固定特性、全体色効果、装備カード8枚選択、装備隣接LINK、永続バックパック
- 保管装備、家宝認定、持込装備、支給品、戦果選択、保険、傷跡
- 複数役職レベル、上級・複合・隠し職、勢力限定職
- 勢力評判、外交係数、勢力関係図、階級能力、情勢フラグ
- キャラクター作成、役職衣装、勢力別拠点、図鑑、記憶庫

このファイルは、ゲーム内の数値と条件を調整するための一覧です。実際の値は [`game.js`](game.js) に定義されています。この資料を変更しただけではゲームには反映されないため、調整時は記載された定義名も同時に変更してください。

## 装備一覧

### 装備個体データ

同じ種類の装備でも別個体として扱います。所持品には装備ID文字列ではなく、次の情報を持つ個体オブジェクトを保存します。

| 項目 | 内容 |
|---|---|
| `uid` | 個体を識別する一意ID。配置や生成カードもこのIDを参照 |
| `templateId` | `ITEMS` に定義された装備種類ID |
| `durability` | 個体ごとの現在耐久度。初期値6 |
| `uses` | その個体から生成されたカードの累計使用回数 |
| `heirloom` | 将来の家宝指定に使うフラグ |
| `evolution` | 将来の個体進化・変質に使うデータ |
| `scars` | 将来の傷・恒久的特徴を記録する配列 |
| `insured` | 将来の保険効果に使うフラグ |

旧形式の装備ID文字列は読み込み時に自動で個体へ変換されます。永続データのスキーマ版は `version: 8` です。装備・拠点・ダンジョン・役職・勢力に加え、作成したキャラクター外見を保存します。

## キャラクター作成と拠点表示

- 初回起動時にキャラクター作成画面を表示します。
- 体格・顔は男性系2種、女性系2種の合計4プリセットです。能力差はありません。
- 各プリセットに短髪、ミディアム、結び髪の3種があります。
- 選択内容は `META.character.body` と `META.character.hair` に保存します。
- キャラクター素材は4列×3行の透過済み `assets/character-creator-sheet.png` です。
- 役職衣装は4体格×4系統の透過済み `assets/role-costume-sheet.png` を素体へ重ねます。

- 拠点中央と戦闘画面のプレイヤー表示で同じ外見を使用します。
- 拠点では現在職に応じた衣装とオーラ、役職名・レベル、勢力階級、選択中の家宝1個を表示します。
- 勢力背景は `assets/faction-hub-sheet.png` の2×2差分です。

形状は左上を `[0,0]` とした相対座標です。回転時は90度ずつ変化します。

### 装備調整時の注意

- 形状は `ITEMS.<ID>.shape` で変更します。
- 配置効果の説明文は `rule`、実際の計算は `deckData()` 内にあります。両方を変更してください。
- `type` は `weapon`、`armor`、`rune`、`supply` のいずれかです。役職・配置効果・隠し職判定に使われます。
- 装備イラスト位置は `ITEM_ART` に定義されています。

### 複数役職レベル

- 役職レベルは `META.jobLevels` に恒久保存されます。
- 初回のみ役職を選択します。現在職は `META.currentRole` に保存され、次回以降は選択画面を飛ばします。
- 役職変更はダンジョン内の「継職の祭壇」でのみ可能です。戦闘勝利と祭壇修練で現在職のレベルを獲得します。
- 基本職と上級職の最大レベルは10、隠し職は15です。
- 全役職を合計したキャラクターレベルは100が上限です。
- 習得済み役職の戦闘効果はすべて同時に適用されます。
- 数値補正の追加成長は3レベルごとです。複数職を重ねても1レベルごとに急激に強くならないよう抑えています。
- 基本職Lv.5以上で対応する上級職を祭壇から解禁できます。転職しても元の役職レベルは失われません。

既存の配置・装備条件に加え、次の役職レベル構成でも隠し職を解禁できます。

| 隠し職 | 役職レベルによる代替解禁条件 |
|---|---|
| 万刃の王 | 戦士Lv.5＋斥候Lv.5 |
| 不動の聖者 | 守護兵Lv.5＋工匠Lv.5 |
| 境界の織手 | 工匠Lv.7＋斥候Lv.3 |
| 盤上舞踏家 | 四基本職がすべてLv.3以上 |

## 協力勢力と階級

鉄殻軍、胞子教団、荷造り師組合は初期解禁です。遠征準備で協力勢力を選び、戦闘勝利ごとに貢献度+1、ボス勝利時は+3を得ます。虚無の巡礼者は専用イベント、または虚ろなる大記憶庫踏破で解禁されます。

| ID | 勢力 | 階級効果 |
|---|---|---|
| `iron` | 鉄殻軍 | 防具カードの防御+階級 |
| `spore` | 胞子教団 | 戦闘勝利後の追加回復+階級 |
| `guild` | 荷造り師組合 | 商店価格を階級×5%割引 |
| `void` | 虚無の巡礼者 | 獲得金を階級×5%増加、呪剣が報酬候補に出現 |

| 階級 | 必要貢献度 | 限定職 |
|---:|---:|---|
| 1 | 0 | なし |
| 2 | 10 | なし |
| 3 | 25 | 勢力ごとの限定職を解禁 |
| 4 | 50 | 最高効果 |

階級3で鉄盟前衛、胞子森導師、荷造商務官、虚無の使徒をそれぞれ解禁します。勢力はジョブとは別の成長軸で、勢力を変更しても過去の貢献度は失われません。

## 商店価格

基本価格は次の式です。

```text
14 + 使用マス数 × 4
```

補正は乗算されます。

- 行商人の鞄：×0.8
- 荷造り師組合：×0.85

例：2マス装備を行商人の鞄＋組合階級3で買う場合、`(14+8)×0.8×0.85=14.96` を四捨五入して15Gです。

### 家宝と傷跡

- 家宝は最大1個を指定します。
- 家宝カードを使った回数と、家宝を伴って勝った戦闘・ボス・ダンジョン、敗北回数を個体履歴へ記録します。

### 道具袋

初期容量は5枠で、開始時は回復薬2個、防護薬、火炎瓶、活力薬を所持します。自分のターン中にカード・エネルギーとは別枠で使用し、使用した個体だけが減ります。回復、防御、直接攻撃、エネルギー、ドローの代表5種類を実装しています。

### 色マス・デッキ選択・装備LINK

バックパックの各マスには火・水・風・土の色があります。装備個体にもセルごとの色と特性があり、入手時に抽選されて個体へ固定保存されます。配置された全装備がカード候補を生成し、その中から最大8枚を装備カードとして選びます。基本職業カード4枚は固定です。装備セルと真下のバックパックマスが同色なら、バックパック全体の一致色数へ加算されます。回転すると形状と装備色が一緒に回転します。

荷造り一覧の小型形状プレビューも、武器・防具などのカテゴリ色ではなく各個体の実際のセル属性色を表示します。回転時は形状と属性色の配置が一緒に回転します。

色一致とは独立して、上下左右に接する装備同士はLINKします。剣と盾、ルーンと武器、結晶と装備などの固有LINK効果があり、剣＋熾火、盾＋結晶、爆弾＋フラスコはカードを追加せず上位カードへ置換します。色一致、複数装備の隣接、8枚の採用カードが競合する構成です。

| 属性条件 | 基本効果 |
|---|---|
| 火1点ごと | すべての攻撃カード+1 |
| 水1点ごと | すべての防御+1、2点ごとに回復+1 |
| 風3以上 | すべてのカードのコスト-1 |
| 土2点ごと | すべてのカードへ防御+1 |

属性は配置中の全装備から集計し、カードを採用していない装備の一致色も全体効果へ寄与します。個体特性は火・水・風・土のいずれかと必要値を持ち、全体色数が条件へ達するとその装備由来カードへ追加効果を与えます。カードの特殊変化は装備同士のLINK条件で発生します。選択した装備カードが8枚未満でも、現在の基本職に応じた4枚の職業カードで戦闘を開始できます。

### 永続バックパックと戦利品袋

- 拠点由来装備の配置・回転は `META.combatPack` に個体UID単位で保存します。
- `META.loadouts` に3つの編成セットを保存し、各セットが `backpack`、`slots`、`deck` を所有します。旧 `META.combatPack` と `META.combatDeck` は選択中セットの実行用データとして同期します。
- 次回遠征では保管庫から該当個体を自動で持ち込み、同じ配置を復元します。
- 敗北しても戦闘バックパック内の装備と配置は維持します。
- ダンジョン報酬と宝物装備は `state.lootBag` へ入り、ラン中の戦闘には使用できません。
- 踏破時に戦利品袋の全装備を持ち帰り、拠点へ戻した時点で色と特性を鑑定表示します。
- 継職の祭壇・回収箱では戦利品袋の全装備を途中鑑定し、保管庫へ送れます。
- 敗北時は未確保の戦利品袋を失います。

### 個体特性

試作では各装備個体へ特性を1個付与します。火・水・風・土それぞれに必要色5と7の特性があり、攻撃、防御、コスト、ドロー、山札回帰、耐久消費無効を付与します。装備種類、形状、基本カード、基本LINKは固定で、セル色と特性だけが個体差です。

### 戦闘状態効果の共通形式

カードと敵行動は共通の `effects` 配列を持ち、`type`、`amount`、`duration`、`target` だけで状態を付与します。定義と割当は `content-data.js`、評価処理は `game.js` に分離しました。現在の代表実装は強化、弱体、脆弱、毒、火傷、再生、防具破壊です。ダメージ、ブロック、ターン終了処理、蓄積、残りターン、戦闘終了時リセットは共通関数で処理します。

- 火花：敵へ火傷2を2ターン。
- 酸液：敵へ毒3。
- 堅牢：自分へ再生2を2ターン。
- 胞子の魔術師：毒を付与。
- 鎧喰い虫：防具破壊を付与。
- 虚ろな騎士：脆弱を付与。
- 鉄殻の番兵：自身へ攻撃強化を付与。

戦闘UIはプレイヤー・敵双方の状態名、蓄積値、残りターン、説明を表示します。カードと敵意図には付与予定の状態を表示し、敵意図には現在の強化・弱体・脆弱を反映した予測ダメージも表示します。

### 古塔の俯瞰地図試作

古塔だけは均等な6×6表示をやめ、俯瞰背景上へ36地点を不規則配置します。主塔、正門、石橋、昇降機、兵舎、鍛造所、礼拝所、武具庫、監視塔などの固有建築物は固定位置を持ちます。敵、商人、戦利品、霧は再入場ごとに変化します。地図背景は `assets/old-spire-map.png` です。この地図1枚を1攻略区域とし、製品版では同一テーマの4～5区域をつないで古塔1ダンジョンを構成します。

古塔の第1区域だけに、警戒・崩壊・侵蝕の三角状態を試験実装しています。各値は `-15～+15`、開始値は0です。移動、偵察、壁破壊、制圧、補強、封印、痕跡隠蔽、異界反応の吸収、敵誘引によって複数軸が交換的に動きます。`±5・±9・±13・±15`を段階境界とし、単独限界、2軸+10、全軸±13で一度きりの状態イベントが発生します。発見済みイベントは `META.dungeonDiscoveries` へ恒久記録します。

- 警戒：敵駒の移動頻度を変化させます。
- 崩壊：破壊・修復行動と隠し経路へ接続します。
- 侵蝕：+5以上で敵攻撃、+9以上で敵HPも強化します。
- 三角グラフは中心が-15、中間が0、各頂点が+15です。

## バランス調整の推奨手順

1. まず基礎カードのダメージ・防御・コストを調整する。
2. 次に配置効果なしの初期装備で各敵を試す。
3. 配置シナジーを最大化した場合の上振れを確認する。
4. 役職Lv.1とLv.3、上級職で同じ戦闘を比較する。
5. 契約を複数付けた場合の報酬倍率と難易度を確認する。
6. 恒久強化が最大の状態でもボスが数ターンで無条件撃破されないか確認する。

### 現在の注目点

- 0コストの《刺突》《受け流し》《整流》は、役職や結晶で強化されると連打性能が高くなります。
- 工匠系役職＋星喰い爆薬は大きく上振れするため、ボス部位報酬との組み合わせを確認してください。
- 城塞騎士は防御カードすべてへ補正するため、レベル上昇時に無敵化しやすい構造です。
- 焦燥＋封印契約は獲得金1.95倍になるため、商店経済が崩れないか確認してください。
- 呪剣は配置必須・取り外し不可・隣接コスト増加があります。実際の不利益が十分か確認してください。
- 部位攻撃中は本体HPを削らないため、部位報酬がリスクに見合うか確認してください。
