# PACKSPIRE ゲームデータ・バランス調整資料

このファイルは、ゲーム内の数値と条件を調整するための一覧です。実際の値は [`game.js`](game.js) に定義されています。この資料を変更しただけではゲームには反映されないため、調整時は記載された定義名も同時に変更してください。

## 基本ルール

| 項目 | 現在値 | 主な定義・処理 |
|---|---:|---|
| 基本最大HP | 42 | `metaMaxHp()` |
| 基本初期ゴールド | 24G | `metaGold()` |
| バックパックサイズ | 6×4マス | `renderPack()` / `canPlace()` |
| 基本エネルギー | 毎ターン3 | `startBattle()` / ターン終了処理 |
| 初期手札 | 5枚 | `draw(5)` |
| 最低デッキ枚数 | 4枚 | `startBattle` ボタン処理 |
| 戦闘中整理 | 1エネルギー、各ターン1回 | `organizePack` |
| 装備の基本耐久度 | 6 | `state.durability` |
| 破損時 | 攻撃・防御効果が半減 | `deckData()` / `playCard()` |
| 野営地 | HPを最大12回復、全装備を修理 | `enterRoute('rest')` |
| 通常戦闘後回復 | 3HP | `winBattle()` |
| 階層数 | 3階層 | `state.floor` |

### 装備調整時の注意

- 形状は `ITEMS.<ID>.shape` で変更します。
- 配置効果の説明文は `rule`、実際の計算は `deckData()` 内にあります。両方を変更してください。
- `type` は `weapon`、`armor`、`rune`、`supply` のいずれかです。役職・配置効果・隠し職判定に使われます。
- 装備イラスト位置は `ITEM_ART` に定義されています。

### 共通の敵処理

- 敵はターン終了時、65%の確率で配置中の装備1個を妨害します。
- 妨害された装備のカードは次ターンにコスト+1されます。
- 鉄殻軍支配時は敵の防御行動+3です。
- 胞子教団支配時は敵の最大HP+5です。
- 盲目の契約中は奇数ターンの行動予告が隠れます。

## 商店価格

基本価格は次の式です。

```text
14 + 使用マス数 × 4
```

補正は乗算されます。

- 行商人の鞄：×0.8
- 荷造り師組合：×0.85

例：2マス装備を行商人の鞄＋組合支配時に買う場合、`(14+8)×0.8×0.85=14.96` を四捨五入して15Gです。

## バランス調整の推奨手順

1. まず基礎カードのダメージ・防御・コストを調整する。
2. 次に配置効果なしの初期装備で各敵を試す。
3. 配置シナジーを最大化した場合の上振れを確認する。
4. 役職Lv.1とLv.3、上級職で同じ戦闘を比較する。
5. 契約を複数付けた場合の報酬倍率と難易度を確認する。
6. 恒久強化が最大の状態でもボスが数ターンで無条件撃破されないか確認する。

### 現在の注目点

- 0コストの《刺突》《受け流し》《整流》は、役職や結晶で強化されると連打性能が高くなります。
- 工匠系役職＋星喰い爆薬は大きく上振れするため、ボス部位報酬との組み合わせを確認してください。
- 城塞騎士は防御カードすべてへ補正するため、レベル上昇時に無敵化しやすい構造です。
- 焦燥＋封印契約は獲得金1.95倍になるため、商店経済が崩れないか確認してください。
- 呪剣は配置必須・取り外し不可・隣接コスト増加があります。実際の不利益が十分か確認してください。
- 部位攻撃中は本体HPを削らないため、部位報酬がリスクに見合うか確認してください。
