# PACKSPIRE これから行うこと

このノートは、現在のゲームシステムを保ちながら、Unity版を「机の上で進行するTRPG」として仕上げるための実装順序をまとめたものです。

実装済み仕様とデータは `GAME_DATA.md`、ゲーム全体の説明は `README.md`、長期的な未実装案は `UNIMPLEMENTED_IDEAS.md` を参照してください。

## 1. 全体方針

- ゲーム全体を、探索者が机上の本、地図、依頼書、カード、駒を操作して進めるTRPGとして見せる。
- バックパック、カード戦闘、役職、勢力、装備育成など、現在の基本システムは維持する。
- すべての画面を無理に本の中へ入れず、用途に合う机上道具へ切り替える。
- 画面遷移は単なる切り替えではなく、道具を机へ置く、紙を貼る、頁をめくる、巻物を広げる演出へ置き換える。
- ゲーム情報と背景イラストを別レイヤーにし、調整でデータを変えても背景を描き直さずに済む構造にする。

### 画面ごとの媒体

| 機能 | 主な見せ方 |
| --- | --- |
| 拠点・目次 | 机上の記録書と周囲の道具 |
| 役職・ステータス・図鑑 | 本の見開き |
| 依頼選択 | 机へ置かれた掲示板または依頼書の束 |
| 受注中の依頼 | 本へ貼り付けた依頼書、栞、封蝋 |
| 遠征準備 | 地図、荷札、カード、バックパック |
| ダンジョン探索 | 机へ広げた巨大地図または巻物と駒 |
| 建物内部 | 地上地図から差し替える小型の内部地図 |
| 戦闘 | TRPG用の戦闘卓、キャラクター駒、カード |
| イベント | 現在の地図を残し、その上へ情景カードと黒い靄を重ねる |
| 将来の判定 | ダイストレイ、ダイス、補正値札 |

## 2. UI基盤を段階的に変更する

現在のUnity版は、単一の `OnGUI` 描画を中心とした試作です。ゲームロジックは流用できますが、商品水準のアニメーション、レスポンシブ配置、重なり順、文字切れ対策を継続するには、表示部分をuGUIまたはUI Toolkitへ段階的に移します。

### 移行方針

1. セーブ、戦闘、バックパック、役職、勢力、ダンジョン生成などのロジックは変更しない。
2. 画面単位でUIを分離し、共通の文字、ボタン、カード、ポップアップ、ツールチップ部品を作る。
3. 新規画面は原則として新UI基盤で作り、既存画面を一つずつ置き換える。
4. 画面遷移用の共通レイヤーを作り、頁めくり、巻物展開、紙の貼り付けを同じ仕組みで制御する。
5. 全画面表示を基準に文字サイズと操作領域を定義し、解像度ごとの縮小上限を設ける。

### 最初に共通化するもの

- 画面タイトル、本文、注釈、数値の文字規格。
- 選択、決定、戻る、危険操作のボタン規格。
- アイテム、役職、敵、依頼、戦闘カードの共通カード枠。
- 選択中、未発見、使用不可、条件達成の表示。
- 確認ダイアログ、空状態、ツールチップ。
- 机、本、巻物、掲示板を切り替える画面遷移管理。

## 3. 依頼を受けて本へ貼る演出

実装可能。依頼内容そのものは通常のデータとして保持し、見た目だけを机上の紙として扱います。

### 基本の流れ

1. 拠点で「依頼」を選ぶ。
2. 本が少し奥へ下がり、机上へ掲示板または依頼書の束が置かれる。
3. 依頼書を選ぶと、右側へ依頼人、目的、報酬、期限、勢力への影響、推奨情報が展開される。
4. 受注すると依頼書が持ち上がり、記録書の依頼頁へ移動する。
5. 紙が糊、留め具、封蝋などで頁へ貼られる。
6. 遠征地図上に目的地、関係人物、探索ヒントが追加される。
7. 達成後は依頼書へ完了印が押され、報告時に報酬を受け取る。

依頼書の紙、封蝋、印、所属勢力の色を差し替えることで、少数の素材から多くの依頼差分を作れます。依頼文や報酬値は画像へ焼き込まず、UIテキストとして重ねます。

## 4. 巨大な連続マップへ変更する

現在の5区域を順番に攻略する方式は廃止候補とし、「巨大な地上マップ＋建物内部などのサブマップ」へ移行します。

### 推奨構造

- 画面には巨大マップの一部だけを表示する。
- プレイヤーの駒を動かすとカメラが滑らかに追従する。
- 見た目は自由な地図だが、内部処理はノードと接続によって管理する。
- 道、橋、崖、建物、入口は最初から背景に描く。
- 移動ノード、選択表示、敵、イベント、隠し道、警戒範囲はUnity側で重ねる。
- 建物へ入ると、対応する内部サブマップへ切り替える。
- 旧「区域」は攻略順ではなく、城下、外郭、中央塔、地下水路などの地理的な地域として残せる。
- 警戒、崩壊、侵蝕、解明、役職、装備、勢力によって経路や施設の状態を変化させる。

探索マップとカード戦闘は分離します。巨大マップ上で敵に接触したら現在のカード戦闘卓へ移り、射程やリアルタイム操作を新たな必須要素にはしません。

### 必要な補助UI

- 現在地周辺を表示する追従カメラ。
- 探索済み範囲を示すミニマップ。
- 全体地図を確認する専用表示。
- 移動予定経路と到達不能理由の表示。
- 地上と建物内部を行き来するパンくず表示。
- 発見済み施設、依頼目的、勢力支配、危険地域の表示切り替え。
- 一度訪れた安全地点へのファストトラベル。導入時期と制限は要調整。

## 5. マップ画像の制作手順

画像を先に作ってゲームを合わせるのではなく、ゲームとして成立する設計図を先に確定します。

1. 道、建物輪郭、入口、高低差、橋、隠し道、ノードを単純図形で配置する。
2. Unity上で仮マップを動かし、移動、カメラ、分岐、入口を検証する。
3. 確定した白地図を参照画像として、形状を維持したままファンタジー背景へ描き込む。
4. 巨大マップを重なりのある複数区画へ分けて高精細化する。
5. 区画の継ぎ目、道、入口を部分編集で修正する。
6. 背景をUnityへ戻し、ノード座標と操作判定を重ねて最終確認する。

背景へ焼き込むのは道、建物、扉、橋、崖、樹木、地形、ランドマークまでとします。文字、ノード、接続線、駒、選択表示、敵、イベント、警戒範囲は別レイヤーにします。

## 6. ダイスを将来追加する場合

机上TRPGの演出と非常に相性がよい一方、すべてをランダム判定にするとバックパック構築の成果が薄れるため、使用場所を限定します。

### 相性がよい用途

- イベントの技能判定。
- 隠し道、罠、交渉、解錠、追跡。
- 大成功と重大な失敗による追加結果。
- 役職、装備、勢力、傷跡によるダイス追加、振り直し、最低値保証。
- プレイヤーが危険を承知で選ぶ任意判定。

### 避ける用途

- 通常攻撃が毎回外れるだけの判定。
- 完成したバックパック効果を無効にする強すぎる運要素。
- 成功確率や補正内容が見えない判定。

実装時は、判定前に基礎成功率、使用する能力、役職補正、装備補正、結果候補を表示します。ダイスは見た目だけの演出ではなく、ビルドによって操作できるゲーム要素にします。

## 7. 実装順序

### 第1段階：現在の試作を安定させる

- 現在の画面で残る文字切れ、重なり、操作不能、状態反映漏れを修正する。
- 既存ゲームシステムとセーブデータの動作を固定する。
- 全画面表示を基準に、最低文字サイズと最低操作領域を決める。
- 開発者用の画面確認パネルを使い、拠点頁、遠征準備、荷造り、保管庫、役職、勢力、図鑑、探索地図、通常戦、ボス戦、報酬、商店、イベント、カード、戦利品を直接確認する。実装済み。
- 開発者パネルでは、全地点表示、テスト戦利品追加、警戒・崩壊・侵蝕の極端値表示も確認する。

### 第2段階：新UI基盤

- 共通文字規格、カード、ボタン、ポップアップを作る。
- 机上レイヤーと画面遷移管理を作る。
- 拠点、目次、役職、図鑑を新UIへ移す。
- 頁めくりは最初は短いフェードと横移動で代用し、レイアウト確定後に本格アニメーションへ置き換える。

進捗：UI Toolkitの共通ルート、文字・ボタン・カード・空状態・確認ダイアログの生成部品、頁・巻物・戦闘卓の仮遷移レイヤーを追加済み。人物作成、拠点・目次、役職、図鑑、保管庫、勢力、遠征準備、荷造り、戦闘報酬、商店、イベント、遠征終了を新基盤へ移行済み。荷造りでは編成切替、バックパック選択、装備配置・回転・取り外し、装備カード選択、保存まで操作できる。探索地図は巻物、戦闘は戦闘卓として専用描画を維持し、新UIとの共通遷移で接続している。下部ナビゲーションは本の右端に挟んだ付箋形式へ変更済み。

商品化UIの第一巡として、拠点メニューと付箋へ専用ナビアイコンを割り当て、既存の金属・革製UI素材を新UI Toolkit側へ接続した。頁見出し、主要決定ボタン、選択中バッジ、ホバー・押下反応、短い頁遷移、操作結果トーストを共通化済み。荷造り盤では属性一致セル、配置失敗、選択装備が盤面だけで判別できる。今後の追加画像は、既存素材では表現できない依頼書、封蝋、ダイス、巨大地図用素材へ限定する。

### 第3段階：依頼掲示板の試作

- 仮データ数件で依頼一覧、詳細、受注、記録書への貼り付けを作る。
- 勢力、目的地、報酬との接続を確認する。
- 紙、封蝋、完了印の差分規格を決める。

### 第4段階：巨大マップの機能試作

- 古塔パックスパイアの一部分だけを白地図で作る。
- 分岐道、入れる建物2つ、橋または崖、隠し道、建物内部1つを配置する。
- 駒移動、カメラ追従、ノード選択、建物内外の切り替えを実装する。
- 5区域依存の進行処理を、目的地、地域状態、ボス条件へ置き換える。

### 第5段階：マップ美術化

- 試作白地図を基準に背景画像を制作する。
- 道と入口が一致するまで部分修正する。
- 地図を複数区画へ分割し、Unity上で継ぎ目を確認する。
- 世界観、視点、光源、密度、色調をスタイルガイドとして固定する。

### 第6段階：演出追加

- 頁めくり。
- 巻物展開。
- 依頼書の貼り付けと完了印。
- 駒移動とカメラ追従。
- イベントの靄、カード出現、封蝋。
- 必要になった段階でダイストレイとダイス判定を追加する。

## 8. 試作完了の判定基準

- 1280×720以上の全画面表示で主要文字が無理なく読める。
- 依頼を受け、記録書へ貼り、目的地へ移動し、報告する流れが一続きで動く。
- 巨大マップ上で、道と駒の移動が視覚的に一致する。
- 建物へ入り、内部マップを探索して地上へ戻れる。
- 警戒、崩壊、侵蝕と隠し経路が巨大マップ上で機能する。
- 背景画像を変更せずに、ノード、報酬、敵、依頼目的を調整できる。
- 既存の荷造り、装備カード、戦闘、戦利品、永続成長が維持される。
